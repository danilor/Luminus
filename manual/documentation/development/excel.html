Para los casos donde es necesario analizar archivos de excel, el sistema proporciona un módulo que facilita el análisis de dichos archivos, casi de la misma manera en que se suben archivos.

Para realizar este proceso, es necesario que se ejecute el mismo proceso de subida de archivos, con la distinción de que se envía el archivo a análisis en vez de subirlo (Para futuras referencias, el análisis también se puede realizar sobre archivos ya almacenados en el sistema).

$FileAttempt = new FileAttempt( "file" , [ "xls" , "xlsx" ] , 2000000 , "excel_" ); // Max file size its 2 Mb
 
try{
  $File = $this->fileManager()->analyzeFileExcel( $FileAttempt );
}catch (\Error $err){
  $redirect -> setUrl( $this->url( "showupload" ) );
  $redirect -> addError( "Hubo un error al subir el archivo" );
  $redirect -> addError( $err->getMessage() );
  $redirect -> setInput( $Request->getInput() );
  return $redirect;
}catch(\Exception $err){
  $redirect -> setUrl( $this->url( "showupload" ) );
  $redirect -> addError( "El archivo no pudo ser subido debido a que no cumple con los requerimientos descritos" );
  //$redirect -> addError( $err->getMessage() . $err->getFile() . $err->getLine() );
  $redirect -> setInput( $Request->getInput() );
  return $redirect;
}


En caso de error, naturalmente se devuelve a la página principal con la explicación. Sino existe error, entonces se puede utilizar el archivo devuelto $File (App\Classes\Util) y presentar la información o tratarla como sea necesario.

En este ejemplo se puede presentar la información como una tabla en el sistema (utilizando una vista) de la siguiente manera:

<table class="table table-striped table-hover">
  <thead>
   <tr>
    @foreach( $File->getkeys() AS $key )
     <th>{{ strtoupper($key) }}</th>
    @endforeach
   </tr>
  </thead>
  <tbody>
   @foreach( $File->getRows() AS $key => $row )
    <tr>
     @foreach( $File->getkeys() AS $key )
      <td>{{ $row -> $key }}</td>
     @endforeach
    </tr>
   @endforeach
  </tbody>
</table>
