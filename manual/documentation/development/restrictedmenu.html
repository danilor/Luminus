Luminus permite a los módulos el asignar permisos específicos para los menús al igual que para los componentes de cada módulo. Por ejemplo, el modulo requiere que en el menú o navegación aparezca un determinado ítem solamente a los usuarios que sean administradores o que posean todos o alguno de los trabajos que se necesitan.

Para lograr esto, el objeto DMOduleMenuItem posee una serie de métodos que permiten establecer estos permisos. A continuación se detallan los métodos más importantes.

public function setUserAdministrator(bool $UserAdministrator)
Este método lo que hace es establecer si para mostrar el ítem del menú es necesario que el usuario sera administrador. De manera predeterminada este valor siempre es falso, pero permite modificarlo mediante el método descrito.

public function addJob_AllJobs(string $j)
Este método añade un trabajo a la lista de "Todos los trabajos por validar". Es decir, si este método es llamado por ejemplo con los valores de "editor" y "periodista", el sistema antes de mostrar el ítem del menú validará de que el usuario posea AMBOS TRABAJOS O ROLES al mismo tiempo. Con que solamente no posea alguno de ellos el ítem no será mostrado.

public function addJob_AnyJobs(string $j)
Este método añade un trabajo a la lista de "Cualquiera de los trabajos por validar". Es decir, si este método es llamado por ejemplo con los valores de "editor" y "periodista", el sistema antes de mostrar el ítem del menú validará de que el usuario posea CUALQUIERA DE LOS TRABAJOS O ROLES. Con que solamente posea alguno de ellos el ítem será mostrado.

Para los dos casos anteriores (AllJobs y Anyjobs) si el usuario es administrador, automáticamente dará validez (es decir, que un usuario administrador no tendrá ninguna restricción real).

Lo mismo funciona exactamente igual para los objetos DModuleMenu y DModuleNavigation, que con estas funciones descritas se pueden ocultar completamente.
