Luminus, y su sistema de control de tablas y base de datos, permite además el manejo de indices llaves foráneas entre tablas.

Las funciones para añadir llaves foráneas e índices pertenecen al DModuleTable y son las siguientes:


Llaves Primarias

El sistema permite que se indiquen cuantas llaves primarias sean necesarias. Para ello, se deben indicar para la tabla y no para cada campo. Se tienen dos funciones para las llaves primarias perteneciente al objeto DModuleTable que son:

- Añadiendo una llave primaria a la vez (puede ser llamado N numero de veces para N número de llaves primarias.
$table -> addPrimaryKey( "key" );
ó
$table -> addPrimaryKeyToDrop( "key" );
Es necesario entender que el DROP del primary key funciona con el nombre que le proporciona la base de datos a la llave primaria. Antes de enviar una instrucción para borrar la llave primaria es necesario conocer su nombre.

- Establece inmediatamente todas las llaves primarias que se desean usar para esta tabla.
$table -> setPrimaryKeys( ["key"] );


Campos Únicos

También se permite por parte del sistema crear campos unique en la base de datos. Para ello es necesario indicarle al objecto DModuleTable que campos se desean de usar como campos únicos.

$table -> addUniqueField( "key" , "unique_key" );
El primer dato corresponde al campo (field) que se desea usar, y el segundo al nombre del índice que se está creando para este caso. Esta función puede ser llamada N número de veces para añadir N cantidad de campos únicos en el sistema.

NOTA: establecer como campos únicos llaves primarias o índices puede causar problemas de tipo: índice duplicado. El sistema suprime la mayoría de estos errores, especialmente para las actualizaciones, así que es responsabiliadd del programador el verificar la creación de las tablas para la base de datos.


Índice

addIndex( string $index_column , string $index_name )

addIndexToDrop( string $index_name )

Para añadir el íncide es suficiente con indicar el nombre de la columna y el nombre que se le quiere dar al índice. Se pueden añadir cuantos índices se requieran sin ningún problema.

Además, se cuenta con la función para borrar índices que ya no se necesiten en el caso de realizar una actualización al módulo.

NOTA: Si se tienen campos como "unique", automáticamente son convertidos en "index" para la base de datos. Realizar un "unique" y un "index" a un mismo campo puede causar errores.

$table -> addUniqueField( "key" , "unique_key" );
$table -> addIndex( "key" , "key" );


Llave Foránea

addForeign( $columns , string $name , string $foreign_table , string $foreign_column , bool $own = true )

addForeignToDrop( string $name )

Para añadir llaves foráneas es necesario llamar a la función addForeign de la clase DModuleTable. Con esta función se indica la o las columnas (como se puede apreciar, el parámetro de columnas no tiene tipo debido a que puede ser tanto un string como un arreglo para cuando se agregan múltiples columnas a la llave), el nombre de la relación, la tabla con la cual se desea llevar a cabo, y la columna foránea. El último parametro de own lo que indica es si la tabla que estamos indicando para el receptor de la llave foránea ($foreign_table) pretence al módulo o es un módulo o tabla distinta. Con eso se permite realizar relaciones con tablas que no necesariamente pertenecen al módulo (para el caso de querer usar tablas distintas a las del módulo, debe indicarse el último parámetro como false).

De igual manera que los índices, las llaves foráneas pueden ser removidas por motivos de alguna actualización del módulo con solo indicar el nombre de la relación.


Este sistema también soporta las actualizaciones de los módulos por versiones tal y como se ha manejado anteriormente. Si se necesita actualizar un módulo, el proceso es exactamente igual (siendo para añadir o para borrar) que el añadir columnas nuevas a las tablas.
