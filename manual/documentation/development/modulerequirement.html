En ocaciones es necesario que ciertos módulos se relacionen con otros, e incluso, otros módulos pueden pasar a ser requerimientos iniciales de los primeros.

El sistema permite en la definición de cada módulo, una estructura de requerimientos para que, al instalarse, sea necesario la existencia de otros módulos para que éste pueda funcionar.

Para lograr esto, es necesario modificar el archivo de definición (JSON) e incluir los requerimientos. Sea un módulo de requerimiento, o sean varios, el proceso y la estructura son iguales.

{
  "name": "exgallery",
  "label": "Galería Externa",
  "description": "Un módulo para demostrar como funciona el sistema de requerimientos previos de módulos para Luminus. Este módulo tendrá como requerimientos previos dos módulos adicionales: el de galería y el de citas.",
  "version": 0.1,
  "module_requirements": [
  {
   "name": "gallery",
   "min": 0.1,
   "max": 2
  },
  {
   "name": "cutequotes",
   "min": 0.1,
   "max": 2
   }
  ]
}
 

Para facilitar el uso de JSON, existen sitios online que permiten la creación y modificación de estos archivos tales como jsoneditoronline

Una vez creada la definición, al momento de instalar o actualizar el módulo, el sistema realizará la verificación correspondiente de los requerimientos del módulo.

Este requerimiento hará que para la instanciación del módulo, se carguen de manera adicional los módulos requeridos y sean enviados al módulo principal por medio del Request. Además, el Request suministra un método de fácil acceso a estos módulos por medio de su nombre.

Por ejemplo, si se quisiera hacer uso de una de las tablas de alguno de los módulos en requerimiento, se podría utilizar la siguiente sentencia:

$Request->extraModule("cutequotes")->table("quotes")

Ésta convocará el módulo de cutequotes, y con ello, todos los métodos relacionados, como el de table().

A continuación se presentan algunos ejemplos de uso de los módulos adicionales.

Y de manera similar, se pueden establecer módulos opcionales con el tag de module_optional en el archivo de definición del módulo. La diferencia con los requeridos es que si no se encuentran, no provocarán un error al cargar o instalar el módulo.

{
 "name": "exgallery",
 "label": "Galería Externa",
 "description": "Un módulo para demostrar como funciona el sistema de requerimientos previos de módulos para Luminus. Este módulo tendrá como requerimientos previos dos módulos adicionales: el de galería y el de citas.",
 "version": 0.1,
 "url": "http://globalex.cr",
 "module_optional": [
 {
  "name": "gallery",
  "min": 0.1,
  "max": 2
 },
 {
  "name": "cutequotes",
  "min": 0.1,
  "max": 2
 }
 ]
}
