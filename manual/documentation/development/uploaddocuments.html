El sistema permite el manejo independiente de archivos para que los creadores de módulos no tengan que preocuparse por el proceso.

El proceso consiste en crear un objeto tipo FileAttempt que va a contener:

  • Nombre del control (attributo name del input)
  • Extensiones Permitidas
  • Tamaño máximo
  • Prefijo de la imagen

Se crea un objeto para cada archivo a subir, y estos archivos se envían al FileManagement del módulo para ver evaluados, verificados y subidos.

$FileAttempt = new FileAttempt( "imagen" , [ "jpg" , "jpeg" , "png" , "bmp" ] , 1.5 , "gal_" );
try{
  $File = $this->fileManager()->attemptFileUpload( $FileAttempt );
  $File -> setTitle( $Request->input( "title" ) );
}catch (\Error $err){
  $redirect -> setUrl( $this->url( "addimage" ) );
  $redirect -> addError( "Hubo un error al subir el archivo" );
  $redirect -> addError( $err->getMessage() );
  $redirect -> setInput( $Request->getInput() );
  return $redirect;
}catch(\Exception $err){
  $redirect -> setUrl( $this->url( "addimage" ) );
  $redirect -> addError( "El archivo no pudo ser subido debido a que no cumple con los requerimientos descritos" );
  $redirect -> addError( $err->getMessage() . $err->getFile() . $err->getLine() );
  $redirect -> setInput( $Request->getInput() );
  return $redirect;
}

Si sucede algún error o alguna excepción, es importante capturarla para mostrar el error indicado. El método de "attemptFileUpload" devuelve un objecto tipo "File" (DModuleFile) que contiene la información del archivo para que el módulo haga con esta lo que sea necesario.

Para finalizar, es necesario ejecutar el complete de la subida de los archivos:

$this -> fileManager() -> completeAttempts( );

Esto es para las ocaciones donde se tiene que subir más de un archivo al mismo tiempo. Así los archivos están siendo subidos y verificados, y si por ejemplo en una lista de 10 archivos para subir, el último falla, entonces los archivos son desechados después de una hora.

Para mantenerlos es necesario el ejecutar el comando anterior, lo cual los "escribirá de forma permanente" en el disco.

Para leer los documentos, el módulo tiene una función especial que le permite, indicando el nombre del archivo, obtenerlo para descargarlo o visualizarlo.
